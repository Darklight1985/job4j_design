Объект юзер состоит из двух полей
int и String

Заголовок объекта User весит = 16 байт

поле int = 4 байта

Заголовок объекта String = 16 байт
3 поля инт, кажде 4 байта. в итоге 3*4 = 12 = байт
Ссылочная переменаа на объект массива  = 4 байт
Сумма 32 байта

Внутри String массив из 7и символов
Заголовок = 16 байт
Данные по длине массива = 4 байта
Примитивы символов 2*7 = 14 байт
Округляем до 40 байт

Итого 16+4+32+40 = 92 байта
Так понимаю итоговое значение так же нужно округлить до 96 байт.

При значении памяти 7 Мбайт  Garbage Collector начинсет удалять объекты.
При этом происходит это еще в процессе создания новых объектов.
Размер памяти под молодое поколение по умолчанию должен тогда состсавлять чуть больше 2Мб
А размер Эдема еще меньше. Видимо при близкости к заполнению GC начинает удалять старые экземпляры класса.
